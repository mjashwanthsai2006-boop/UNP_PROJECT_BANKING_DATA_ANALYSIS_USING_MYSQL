## 📌 Project Overview

I have done this project with collaboration of UNP, focusing on designing and analyzing a **Banking Database System using MySQL**.

This project demonstrates how a real-world banking system can be structured using relational databases. It includes management of **customers, accounts, transactions, and loans**, along with meaningful SQL queries for data analysis.

---

## 🎯 Objective

- To design a structured banking database  
- To implement relationships using **primary & foreign keys**  
- To perform **data analysis using SQL queries**  
- To simulate real-world banking operations  

---

## 🛠️ Technologies Used

- **MySQL**
- **SQL (DDL & DML)**
- Database Concepts (Normalization, Relationships)

---

## 🗂️ Database Structure

The project consists of the following tables:

### 👤 Customers
- Stores customer details like name, email, city, mobile number, and DOB  

### 💳 Accounts
- Contains account type (Savings/Current), balance, and opening date  

### 💸 Transactions
- Records deposits, withdrawals, and transfers  

### 🏦 Loans
- Stores loan amount, interest rate, and loan status  

---

## 🔗 Entity Relationships

- One customer → Multiple accounts  
- One account → Multiple transactions  
- One customer → Multiple loans  

---

## 📊 SQL Queries Implemented

- Retrieve all customers  
- Join customers with accounts  
- Find customers with balance > 20,000  
- Calculate total bank balance  
- View transactions of a specific account  
- Count account types  
- Identify loan holders  
- Find highest balance account holder  
- Analyze deposits & withdrawals  
- Filter active loans
